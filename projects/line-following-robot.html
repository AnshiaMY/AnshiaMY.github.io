Autonomous Line-Following Robot

A sensor-based robot designed to follow black paths using Arduino control logic, motor response, and repeated testing across different track conditions.

Arduino C++ Sensors Motor Control Robotics Testing
View Demos Final Reflection

Project Overview

The Build

This project combined mechanical assembly, wiring, programming, sensor testing, and debugging to create a robot that could respond to different path conditions.

What I Built

An autonomous robot that detects black paths and adjusts its movement to stay aligned with the track.

My Role

I worked on programming, wiring, sensor calibration, testing, debugging, and observing how the robot responded during different trials.

Tools Used

Arduino, C++, IR sensors, DC motors, motor driver, chassis components, wiring, and physical test tracks.

Testing the Robot

The Demos

I tested the robot in three different track conditions to observe how it handled basic tracking, curved paths, and boundary-style movement.

01

Straight Line Test

This was the baseline test. I used it to check whether the robot could maintain stable movement on a simple black path before testing more complex track shapes.

  • Checked basic sensor response
  • Observed motor correction behaviour
  • Used as a starting point before curved-path testing
02

Racetrack Path Test

This test challenged the robot’s ability to handle curves and direction changes. It helped show how well the control logic responded when the robot needed to turn.

  • Tested curved-path tracking
  • Observed overcorrection during turns
  • Helped evaluate movement stability
03

Black Circle Test

This test showed how the robot behaved inside a dark circular boundary. It helped demonstrate how the sensor-based logic responded when surrounded by a black region.

  • Tested boundary-style behaviour
  • Observed how the robot reacted to dark surroundings
  • Showed a different use of the same sensor logic

System Logic

Behind the System

The robot’s behaviour came from the interaction between sensor readings, Arduino logic, and motor response.

1

Sensors Detect Path

The sensors read contrast between the black path and the surrounding surface.

2

Arduino Reads Input

The Arduino processes the sensor readings and determines how the robot should respond.

3

Motors Adjust

The motors change speed or direction to help the robot correct its position.

4

Robot Corrects

The robot moves back toward the path and continues following the track.

Testing & Debugging

What I Improved

Testing helped reveal how small physical and software changes affected the robot’s movement.

Turning was inconsistent

I tested the robot on simpler paths first, then used curved tracks to observe where it overcorrected or lost alignment.

Sensor readings were sensitive

I learned that lighting, surface colour, and sensor placement could affect the robot’s behaviour, so testing on the actual track surface was important.

Physical movement affected the code

I observed that wiring, motor speed, chassis alignment, and sensor position all affected how the robot responded to the same control logic.

Final Takeaway

Final Reflection

This project helped me understand how software, electrical components, and mechanical design interact in a real robotic system. The most valuable part was testing the robot in different conditions and seeing how small physical changes affected the control behaviour.

Future Improvements

  • Cleaner wiring and cable management
  • More secure sensor mounting
  • Smoother turning control
  • More consistent testing data across different track layouts
