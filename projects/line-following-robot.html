Autonomous Line-Following Robot

A Raspberry Pi–based mobile robot designed to track black tape on a white surface using infrared reflectance sensors, PID control, motor response, and iterative testing across line-following and boundary-detection conditions.

Raspberry Pi Python PID Control Infrared Sensors Motor Control Robotics Testing

Technical Snapshot

Platform Raspberry Pi
Sensing 2 infrared reflectance sensors
Control Python PID controller
Build SolidWorks chassis + wiring

Design Process

From concept to control.

This project began as an exploration of feedback-based movement. Instead of programming the robot to follow a fixed route, the goal was to use sensor data to guide its decisions: tracking a black line during path-following tests and detecting a black boundary during containment testing.

01

Concept

The project started with a simple control question: how can a small robot use feedback from its environment to correct its movement in real time? The black tape and whiteboard surface created a clear contrast that the robot could use to make movement decisions.

02

Mechanical Layout

The chassis was designed in SolidWorks across multiple iterations to support the Raspberry Pi, sensors, motors, wiring, and control hardware. The layout had to keep the sensors close to the surface while leaving enough space for stable mounting and wiring.

03

Electronics & Wiring

The electrical system connected the infrared sensors, motor driver, DC motors, and Raspberry Pi through a regulated 5V/3.3V power setup. Copper-board wiring and multimeter checks were used to verify connections and debug the robot across its subsystems.

04

Control Logic

Python code read the two bottom-mounted infrared sensors and adjusted motor response based on the robot’s position relative to the black line. A PID controller was tuned through repeated testing to improve trajectory stability and reduce unstable movement.

Testing

How it performed.

The robot was tested across different path conditions to evaluate line tracking, turning response, and boundary detection.

Straight-Line Test

Tracking the line

The straight-line test focused on the robot’s ability to detect contrast between the black tape and whiteboard surface. The two infrared sensors helped identify when the robot drifted left or right, allowing the motor response to correct its path.

Racetrack Test

Testing turns and response

The racetrack condition tested how well the robot could handle turns while staying aligned with the black path. This helped evaluate whether the PID tuning and motor adjustments were stable enough for more complex movement than a straight line.

Black-Circle Test

Detecting a boundary

The black-circle condition was not a line-following test. It was used to evaluate boundary detection, where the robot had to recognize the edge of a defined area and respond when it reached that boundary.

Iteration

Challenges and improvements.

The most important improvements came from testing how the mechanical layout, wiring, sensors, and code affected each other.

Sensor Alignment

The infrared sensors needed to sit close enough to the surface to detect reliable black-and-white contrast.

PID Tuning

The controller was tuned over repeated trials to reduce unstable movement and improve line-tracking behaviour.

Power & Wiring

Regulated power and wiring connections had to be verified carefully using multimeter-based debugging.

Mechanical Fit

The chassis had to support sensors, motors, wiring, and control hardware without interfering with movement.

Reflection

What this project taught me.

This project helped me understand that robot performance depends on more than code alone. The mechanical layout, sensor placement, wiring reliability, power regulation, and control logic all affected how well the robot behaved during testing. It was one of my first projects where I could clearly see how mechanical, electrical, and software decisions come together in a working mechatronic system.
