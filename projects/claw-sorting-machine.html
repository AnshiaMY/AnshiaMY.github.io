VEX Toy-Sorting Claw Machine

A group VEX robotics project focused on designing, building, and testing an autonomous claw-based sorting machine. The robot used a claw-and-lift mechanism, sensor-based navigation, and C++ control logic to move objects into selected category zones.

Project Overview

From Playroom Mess To Smart Cleanup.

The project began with the idea of turning a repetitive household task into a structured robotics challenge. Toys and small objects often need to be collected and organized after use, so our group designed a VEX robot that could model part of that cleanup process by transporting an object to a selected category area. The goal was not only to build a moving claw mechanism, but to create an autonomous system with a clear input–action–output sequence. Through simple user input, the robot introduced a colour-to-category association, where each selection represented a sorting destination and the robot responded by completing the matching task. This gave the project both a practical focus and an educational layer: the robot demonstrated automated sorting while reinforcing organization, colour recognition, and cause-and-effect behaviour.

Build Stage 01

Mechanical Frame & Drive Base

The robot’s frame and drive base were designed from the VEX IQ kit components provided for the project, rather than from a prebuilt chassis. After the claw and vertical lift concepts were developed, the base was designed around those mechanisms so the robot could support the VEX Brain, drive motors, lift motor, claw motor, sensors, and object-handling system within one stable structure. Because the robot needed to carry an object above the drive platform, the frame had to be wide enough to improve stability and reduce tipping during movement. At the same time, the wheel placement and motor positioning had to support straight-line driving, controlled turning, and alignment with the sorting area. The final frame also needed enough mounting space for the lift mechanism and sensors while remaining compact enough to navigate the sorting path reliably.

Build Stage 02

Claw & Lift Mechanism

The object-handling system combined a two-pronged overhead claw with a vertical lift mechanism. During the design process, our group considered several options for securing the object, including a front-facing claw, a pronged lift, and an overhead hook. After testing an early front-facing claw concept, the design was changed because the object grip was not consistent enough. The overhead claw was selected instead because it allowed the user to place the object more directly into the mechanism, which improved reliability during the sorting sequence. To raise and lower the claw, our group developed a chain-based vertical lift inspired by forklift-style lifting systems. Since the VEX kit did not use hydraulics, the lift was adapted using a motor-driven sprocket, a second sprocket, and a closed-loop chain. The lift motor turned the top sprocket, which moved the chain and allowed the claw assembly to travel vertically. This gave the robot a controlled way to raise the object for transport and lower or release it near the selected category box.

Integrating the claw with the lift also required mechanical trade-offs. Because the chain carried the weight of both the claw and the object, the claw assembly had to be kept as light as possible, and additional supports were added to reduce stress on the plastic VEX components. This made the final mechanism more reliable while still allowing the claw and lift to work together as one object-handling subsystem.

Build Stage 03

Sensor-Based Navigation

The robot used multiple sensors to connect user input with autonomous movement. At the beginning of the sequence, the touch sensor counted one, two, or three taps and converted that input into a selected sorting category. Once the category was chosen, the robot used its colour sensor to scan the floor for the matching coloured stripe, which acted as a path marker for the correct drop-off area. Additional sensors helped make the movement more controlled after the correct stripe was detected. The gyro sensor supported the 90-degree turn toward the category box, while the distance sensor helped the robot stop at a consistent distance from the box wall before releasing the object. Motor encoder feedback was also used to track movement so the robot could return more reliably toward its starting position. Together, these sensors allowed the robot to make decisions based on feedback rather than relying only on fixed timing.

Build Stage 04

Autonomous Sorting Sequence

The robot’s autonomous behaviour was programmed in C++ using VEXcode and was structured around the full sorting cycle rather than a single timed routine. The code was divided into separate functions for each major action, including reading the touch sensor input, opening and closing the claw, raising and lowering the lift, driving forward, scanning for the selected colour stripe, turning toward the box, approaching the drop-off area, releasing the object, and returning to the start. This modular structure made the program easier to test because each subsystem could be debugged before being connected into the full sequence. For example, the touch-sensor function converted one, two, or three taps into a category selection, while the colour sensor routine continuously checked the floor until the matching stripe was detected. Once the stripe was found, a turning function used gyro feedback to rotate the robot toward the box, and a distance-sensor routine helped stop the robot at a controlled distance before releasing the object. The code also had to account for real-world movement errors. Gyro-based turns were not always exactly 90 degrees, and sensor readings could vary depending on alignment, surface reflection, or distance from the target. To improve consistency, the program used calibrated thresholds, encoder-based movement tracking, and repeated testing to tune how the robot moved between each stage. The main coding challenge was coordinating sensor readings, motor commands, and mechanical timing so the physical robot could complete the same sorting sequence reliably across multiple trials.

Troubleshooting

Debugging the mechanism.

Testing revealed that the robot’s performance depended on more than the code. The claw, lift, sensors, chassis movement, and autonomous sequence all had to work together reliably for the sorting task to succeed.

Debug File 01

Claw alignment and grip reliability

Symptom

The claw needed to hold the object securely during movement without dropping it before reaching the selected category box.

Diagnosis

The gripping mechanism depended on both the claw position and the way the object sat inside the mechanism. Small alignment issues could affect whether the object stayed stable while the robot drove or turned.

Adjustment

The group refined the claw setup and tested the mechanism repeatedly so the robot could grip, carry, and release the object more consistently during the autonomous sequence.

Debug File 02

Lift positioning and object release

Symptom

The lift needed to raise and position the claw so the robot could release the object into the correct category box.

Diagnosis

The lift height and timing affected whether the object was released in the correct place. If the mechanism was not positioned properly, the sorting sequence became less reliable.

Adjustment

Testing focused on coordinating the lift motor, claw motor, and movement sequence so the robot could approach the box, position the object, and release it more reliably.

Debug File 03

Sensor-based navigation

Symptom

The robot had to detect the correct marked area and align itself with the selected drop-off box during the sorting sequence.

Diagnosis

The autonomous routine depended on sensor feedback from the touch sensor, colour/optical sensor, distance sensor, gyro sensor, and motor encoders. If one reading was inconsistent, it could affect the robot’s path or final position.

Adjustment

The group tested the sensor logic with the movement sequence to improve how the robot detected floor markers, approached the box, and returned to the starting position.

Final Takeaway

Final reflection

What this project taught me.

This project showed me how mechanical design, sensors, and control logic come together in an autonomous robotic system. The robot’s performance depended on the claw, lift, chassis, sensor placement, and code working as one coordinated system rather than as separate parts.

The most valuable part of the project was seeing how design decisions affected the full sorting sequence. A small issue with claw alignment, lift positioning, or sensor feedback could affect whether the robot picked up the object, moved accurately, released it in the correct location, or returned to the start. This helped me better understand the importance of testing and iteration in mechatronics design.

Future Improvements

Next iteration notes

  • Improve claw alignment to make object gripping more consistent.
  • Refine the lift mechanism for smoother positioning and release.
  • Improve sensor calibration for more reliable navigation and stripe detection.
  • Test the full sorting sequence with a wider range of object sizes and weights.
